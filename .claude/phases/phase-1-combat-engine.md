# Phase 1 — Pure combat engine

Status: **done**. Full local verification green (63/63 tests, lint, format, build);
matches ROADMAP's exit criterion ("get it green before Phase 2 — it's the proof the
architecture holds"). Implemented on branch `phase-1-combat-engine` per the approved plan
at `.claude/plans/sorted-drifting-dream.md` (kept there for the detailed rationale behind
every design decision below; not duplicated here).

## What was built

All new files under `src/engine/` (existing `rng.ts`/`rng.test.ts` untouched):

- `ids.ts`, `types.ts` — branded `CreatureId`; `Creature`, `Action` (one variant,
  `attack`), `CombatState`, and the full `CombatEvent` union (intent + consequence +
  lifecycle, all PascalCase discriminants).
- `config.ts` — `ROUND_CAP` (100, placeholder), affinity multipliers, chip-floor rate.
- `affinity.ts` — the Body>Spirit>Mind>Void>Primal>Body cycle as data.
- `effective-stats.ts` — `getEffectiveStat` (passthrough seam) and `getOffensiveStat`
  (remap-aware seam), both no-ops today, real seams for Phase 3.
- `damage.ts` — `calculateDamage`, the two-pool formula, floored once at the end.
- `turn-order.ts` — `buildTurnQueue`, frozen per round, side→slot→id tie-break.
- `targeting.ts` — `getDefaultTarget`, first-living-enemy-by-slot.
- `phase-hooks.ts` — the six-point no-op hook seam for Phase 3.
- `combat.ts` — `createCombat`/`resolveTurn`/`resolveFight`, decide/execute split for
  `Action` so Phase 2's scripting interpreter only has to replace the "decide" half.
- `__fixtures__/creatures.ts` — test-only `makeCreature`/`makeParty` factories.
- Unit tests for every leaf module, `combat.test.ts` for resolver-level behavior, and
  four golden-replay fixtures (`__golden__/`): 1v1, 6v6, an affinity matchup, and a stomp.

## Plan-review corrections (caught before any code was written)

The design went through an agent-authored draft, my own review, and then the user's
review before implementation started. Two real bugs were caught and fixed at the plan
stage — worth recording since they'd have been easy to ship silently:

1. **Skipped turns must still emit the `TurnStarted`/`TurnEnded` bracket.** The first
   draft returned early with no events at all for a creature that died before its queued
   turn — which directly contradicts the reason those two events were made real log
   entries in the first place (an explicit boundary even for no-op turns). Fixed: the
   bracket is always emitted; only the action and the turn-start/turn-end hooks are
   gated on the actor being alive. An empty bracket *is* the skip signal.
2. **`localeCompare` in the turn-order tie-break is a determinism hazard.** ICU/locale
   behavior can differ between machines (local vs CI), which would silently break
   byte-identical golden-replay output across environments. Fixed: plain codepoint
   `<`/`>` comparison.

Plus two smaller corrections: the round-cap gate is checked *before* starting a new round
(so exactly `ROUND_CAP` full rounds complete, then a clean draw — not a round cut off
after one creature's turn), and the `golden-stomp` fixture's underdog stat is pinned so
its Attack is *at or below* the stomper's Defence (required for `wasChipOnly: true`, not
just "small" damage).

## Golden fixture discipline

Every golden fixture's expected event log was **hand-derived from the formula**, not
generated by running the implementation and copy-pasting its output — per CONVENTIONS'
explicit warning against turning a regression detector into a rubber stamp. In practice
this meant: reasoning through turn order and targeting by hand (mechanical, no ambiguity),
computing each hit with an independent calculator script (`node -e "..."`, plain
arithmetic, never calling `calculateDamage`/`resolveFight`), and — critically — verifying
the *exact* floating-point value JS produces before hardcoding it. This mattered in
practice: `5.15 * 0.75` evaluates to `3.8625000000000003`, not the clean `3.8625` hand
arithmetic would suggest, and `toEqual` requires exact equality. Every fixture number was
checked this way before being written into the `.fixture.ts` files; all four passed
against the real implementation on the first run.

## Deliberately out of scope (per ROADMAP, next phases)

Cast/Defend/Provoke/Wait, any scripting/interpreter, trait/status content and the
hook-dispatch/cascade system itself, real creature/species data, currencies/rewards/XP,
and any combat UI. The seams for most of these (remap-aware OffStat lookup, empty
damage-modifier pools, the six-point phase-hook stub, `Action` as a real dispatched value)
are already in place so Phase 2/3 extend rather than rewrite.
